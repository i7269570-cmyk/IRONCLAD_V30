# 핵심 실행 흐름

```text
1. run.py 시작
2. preflight_gate.py 실행
   → 필수 파일, schema, state, 설정 이상 여부 검사

3. scheduler.py 실행
   → 현재 시간 모드 결정
   → TRADE / NO_ENTRY / FORCE_EXIT / CLOSED

4. data_loader.py 실행
   → 전종목 금지
   → Top-N 30~50개만 로드

5. selector.py 실행
   → Top-N 안에서 최종 3~5개 선정

6. regime_filter.py 실행
   → 시장 상태가 진입 가능한지 판단

7. entry_engine.py 실행
   → 조건식 충족 여부 판단

8. risk_gate.py 실행
   → 최대 2포지션 확인
   → 동일 자산군 중복 금지 확인
   → 총 노출 제한 확인
   → position sizing 계산
   → 통과/차단 결정

9. pre_order_check.py 실행
   → 주문 직전 최종 안전 검사

10. order_manager.py 실행
    → 자산별 API로 주문 실행
    → 재시도/중복 주문 방지

11. fill_tracker.py 실행
    → 체결 상태 반영

12. state_manager.py 실행
    → atomic write로 state 저장

13. ledger_writer.py 실행
    → 거래/체결/청산 로그 기록

14. exit_engine.py 실행
    → TP/SL/시간청산/강제청산 수행

15. position_reconciler.py 실행
    → 실제 계좌와 내부 상태 동기화

16. 예외 발생 시 exception_handler.py 실행
    → Cancel-All
    → incident 저장
    → SAFE_HALT
```

# 시간 구조 최종 고정

```text
09:00 ~ 14:50   → TRADE
                  진입 + 청산 가능

14:50 ~ 15:00   → NO_ENTRY
                  신규 진입 금지
                  청산만 허용

15:00 ~ 15:30   → FORCE_EXIT
                  모든 포지션 강제 청산 구간
                  15:30 전에 반드시 0 포지션

15:30 이후      → CLOSED
                  이미 현금 상태
                  진입/청산 모두 없음
```

핵심은 이것입니다.

```text
15:30에 청산하는 구조가 아님
15:00부터 청산 시작
15:30 전에 이미 전량 현금화 완료
```
